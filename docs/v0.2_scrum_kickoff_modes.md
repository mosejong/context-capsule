# v0.2 Scrum Notes and Project Kickoff Modes

v0.2 expands Context Capsule from single-task handoff packets into meeting and project-start planning packets.

The goal is not to evaluate teammates or assign work automatically. The goal is to turn meeting text into clear notes, decisions, action items, issue drafts, and team-lead review prompts.

v0.2.1 adds Project Health Check as a third collaboration mode. It estimates MVP/prototype readiness from meeting text and asks what needs to be clarified before the next work session.

## Scrum Notes Mode

Input:

- scrum notes
- meeting transcript
- instructor feedback
- optional project context

Output:

- source summary
- decisions
- blockers
- direction changes
- next actions
- open questions
- role-discussion questions
- GitHub Issue drafts
- team-lead notes
- safety notes

CLI:

```powershell
.\context_capsule_cli.bat scrum-notes --text "Coach: Reduce MVP scope. Team: Build release notes." --json
```

Save Markdown:

```powershell
.\context_capsule_cli.bat scrum-notes --text-file .\meeting.txt --project-context "3rd project" --save --json
```

Fixture demo:

```powershell
.\context_capsule_cli.bat scrum-notes --text-file .\tests\fixtures\scrum_runtime_issue_ko.txt --project-context "KDT beta project" --json
```

## Project Kickoff Mode

Input:

- contest or project topic
- idea meeting notes
- deadline
- constraints
- self-reported team context

Output:

- one-line pitch
- MVP scope
- out-of-scope list
- workstreams
- risks
- open questions
- role-discussion questions
- GitHub Issue drafts
- submission checklist
- team-lead notes
- safety notes

CLI:

```powershell
.\context_capsule_cli.bat kickoff --topic "Scrum-to-execution planning tool" --notes "Build Scrum Notes Mode first. Discord API later." --deadline "2 weeks" --json
```

Save Markdown:

```powershell
.\context_capsule_cli.bat kickoff --topic "3rd project" --notes-file .\ideas.txt --deadline "2 weeks" --save --json
```

Fixture demo:

```powershell
.\context_capsule_cli.bat kickoff --topic "Context Capsule v0.2" --notes-file .\tests\fixtures\project_kickoff_context_capsule_ko.txt --deadline "2 weeks" --json
```

## Project Health Check

Input:

- project status text
- scrum notes
- kickoff notes
- optional deadline
- optional self-declared scope

Output:

- MVP readiness percentage
- prototype readiness percentage
- stability label
- missing meeting items
- next meeting questions
- ownership check

CLI:

```powershell
.\context_capsule_cli.bat health --text "v0.2 UI done. pytest passed. 주말 재테스트 전 README 정리." --my-scope "README, UI" --json
```

Project Health Check is not teammate scoring and not automatic assignment. It is a meeting-preparation aid.

## Role Discussion Questions

v0.2 can suggest questions for a team lead to ask, but it must not assign people.

Examples:

```text
Who wants to take the first small task based on self-reported capacity?
Which task needs pairing, review, or instructor help before someone takes it?
Which task should stay unassigned until the team lead confirms scope?
```

These questions are planning prompts, not evaluation results.

## Safety Boundary

Allowed:

- summarize meeting notes
- extract decisions and blockers
- create action items
- create issue drafts
- recommend what the team lead should ask next

Not allowed:

- score teammates
- infer skill from speaking volume
- auto-assign owners
- analyze hidden meeting recordings
- submit or deploy automatically

Team members should self-report capacity, preferred work, and blockers. Final assignment stays with the human team lead.

## Product Position

Short version:

```text
Turn scrum notes and kickoff ideas into reviewable work packets.
```

Portfolio version:

```text
Context Capsule v0.2 helps teams move from meeting discussion to execution by extracting decisions, risks, next actions, and issue drafts while keeping people evaluation and final assignment under human control.
```
