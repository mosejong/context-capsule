# KDT Tester Invite Packet

Context Capsule을 KDT 수강생이나 팀원에게 테스트 부탁할 때 보내는 1장짜리 운영 문서입니다.

목표는 칭찬을 받는 것이 아니라, **처음 쓰는 사람이 어디서 헤매는지** 빠르게 모으는 것입니다.

## 1. Discord에 바로 보낼 문구

```text
Context Capsule v0.2.15 베타 테스트 부탁드립니다.

이 도구는 AI에게 바로 "고쳐줘"라고 시키기 전에,
내 프로젝트에서 어떤 파일을 봐야 하는지, 무엇을 건드리면 안 되는지,
AI에게 어떤 프롬프트로 넘기면 좋은지 정리해주는 로컬 도구입니다.

다운로드:
https://github.com/mosejong/context-capsule/releases/latest

테스트 방법:
1. ZIP 다운로드
2. 압축 해제
3. START_HERE_KO.md 먼저 보기
4. run_context_capsule.bat 더블클릭
5. http://localhost:8501 열기
6. 자기 프로젝트 폴더 경로를 넣고 아래 문장 중 하나를 입력

추천 입력:
- 리드미 손보자
- 로그인 안돼
- 로컬 실행 안돼
- README 포트폴리오용으로 다듬어줘
- auth는 건드리지 말고 문서만 바꾸자
- 이거 왜그래?

보고 싶은 것:
- 실행이 막히는지
- 어디를 눌러야 하는지 헷갈리는지
- 근거 파일이 기대한 파일과 맞는지
- 충돌/위험 설명이 이해되는지
- 복붙 프롬프트가 실제 AI에게 넘길 만한지
- 다시 쓸 의향이 있는지

좋게 말 안 해도 됩니다.
"뭐 누르라는 거임?", "결과가 너무 많음", "내가 기대한 파일이 아님" 같은 피드백이 제일 필요합니다.

주의:
.env, API key, GitHub token, 개인정보, 회사/팀 비공개 코드는 보내지 말아주세요.
```

## 2. 10분 테스트 순서

| 시간 | 할 일 | 확인할 것 |
| --- | --- | --- |
| 1분 | ZIP 다운로드, 압축 해제 | 파일 목록에서 `START_HERE_KO.md`와 `run_context_capsule.bat`가 보이는가 |
| 2분 | `run_context_capsule.bat` 실행 | 브라우저에서 `localhost:8501`이 열리는가 |
| 2분 | 자기 프로젝트 경로 입력 | 경로 입력 위치가 명확한가 |
| 2분 | 작업 요청 입력 | 평소 AI에게 말하듯이 적어도 되는가 |
| 2분 | 결과 확인 | `요약 -> 추천 첫 행동 -> 근거 파일 -> 충돌/위험 -> 복붙 프롬프트` 흐름이 이해되는가 |
| 1분 | 피드백 작성 | 기대 파일과 실제 파일, 헷갈린 점을 남길 수 있는가 |

## 4. 테스트 요청 범위

처음 테스터에게는 모든 기능을 다 부탁하지 않습니다.

우선순위:

1. `AI에게 작업 맡기기`
2. `근거 파일`이 맞는지 확인
3. `충돌/위험` 설명이 과하거나 부족하지 않은지 확인
4. `복붙 프롬프트`가 실제 Claude, Codex, ChatGPT에 넘길 만한지 확인

여유가 있는 테스터에게만 추가로 부탁합니다.

- `회의록 정리하기`
- `프로젝트 시작 준비하기`
- `준비도 점검`
- `피드백 모아보기`

## 5. 피드백을 받을 때 주의할 점

좋은 피드백:

- "README.md가 나올 줄 알았는데 하위 README만 나왔어요."
- "로딩 중인지 멈춘 건지 처음엔 몰랐어요."
- "Risk가 너무 많아서 뭘 봐야 할지 모르겠어요."
- "복붙 프롬프트는 좋았는데 근거 파일이 너무 많아요."
- "이 문장으로 물어봤더니 질문만 하고 진행하지 않았어요."

덜 유용한 피드백:

- "좋아요."
- "괜찮은 것 같아요."
- "AI가 해주는 거 아닌가요?"

이런 반응은 다시 질문합니다.

```text
어떤 입력을 넣었고, 어떤 파일이 나오길 기대했나요?
결과에서 가장 먼저 본 부분은 어디였나요?
다시 쓴다면 어떤 상황에서 쓸 것 같나요?
```

## 6. 운영자 체크리스트

- [ ] 최신 Release URL을 보냈다.
- [ ] `START_HERE_KO.md`를 먼저 보라고 말했다.
- [ ] 터미널은 선택사항이라고 말했다.
- [ ] secret, token, `.env`를 보내지 말라고 말했다.
- [ ] 칭찬보다 헷갈린 점이 필요하다고 말했다.
- [ ] 피드백을 `outputs/feedback` 또는 별도 문서로 저장했다.
- [ ] 반복되는 실패 문장을 user-speech QA 후보로 기록했다.
