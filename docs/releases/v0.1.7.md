# Context Capsule v0.1.7

## Release Asset

```text
context-capsule-v0.1.7.zip
```

## Release Type

KDT beta retrieval tuning patch.

v0.1.7 is based on beta tester feedback after v0.1.6. The core v0.1.5 fixes held up: Korean requests can now find English codebase files, and secret/prompt-injection redaction blocks unsafe handoff packets. This release tunes the remaining non-blocking pain points around documentation-only and local-run troubleshooting requests.

## What's Improved

- Documentation-only requests now exclude unrelated code/test chunks before risk analysis.
  - Example: `auth는 건드리지 말고 문서만 바꾸자`
  - Expected result: README/docs are retrieved, protected `auth` is recorded, unrelated code does not become the first target.
- Local-run troubleshooting requests now prioritize launcher/runtime files:
  - `run_context_capsule.bat`
  - `scripts/run_dashboard.ps1`
  - `scripts/install_windows.ps1`
  - `docs/local_app.md`
  - `docker-compose.yml` when present
- Launcher intent now filters out test files and unrelated README/report docs that previously created noisy risk findings.
- The local launcher alias no longer forces `app/main.py` as a mandatory file hint.
- Indexed and hybrid retrieval now share the same intent-aware exclusion rules as keyword retrieval.
- Added regression tests for:
  - security-heavy documentation-only requests not becoming `BLOCKED`
  - local-run requests returning launcher/docs/config before app code

## Beta Feedback Addressed

Before:

```text
auth는 건드리지 말고 문서만 바꾸자
-> code file could rank first in security-heavy repos
-> redacted secret context could cause BLOCKED even though the user asked for docs only

로컬 실행 안돼
-> app/test files could appear before launcher docs/scripts
```

After:

```text
auth는 건드리지 말고 문서만 바꾸자
-> README/docs only
-> auth remains protected
-> auto-start can stay allowed when no unsafe target remains

로컬 실행 안돼
-> run_dashboard.ps1 / install_windows.ps1 / local_app.md / run_context_capsule.bat
-> no app/test file in the first launcher result set
```

## Known Limits

- This is still heuristic intent-aware retrieval, not a full semantic ranking model.
- Security-heavy repositories can still produce MEDIUM mention-risk findings when relevant docs mention credentials, auth, deploy, or token topics.
- Real secret-like values and prompt-injection-like lines remain strict BLOCKED cases.
- First install still needs internet access for dependencies.
