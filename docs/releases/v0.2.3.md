# Context Capsule v0.2.3

v0.2.3 adds Workflow Graph Trace for Work Handoff packets.

This release improves trust and debuggability. Users can now see the local step path that produced a packet instead of only seeing the final result.

## Download

```text
context-capsule-v0.2.3.zip
```

Download the latest ZIP from:

```text
https://github.com/mosejong/context-capsule/releases/latest
```

Then extract and double-click:

```text
run_context_capsule.bat
```

## What Changed

### Workflow Graph Trace

Work Handoff results now include a `작업 흐름` tab.

It shows:

```text
scan_repository
-> understand_request
-> retrieve_context
-> analyze_risk
-> generate_packet
-> review_gate
-> save_output
```

Each step records:

- status: completed, skipped, blocked, or needs input
- summary
- evidence
- next action

### Better Trust Signals

The graph makes these cases clearer:

- normal request completed
- vague request stopped for one clarification question
- risky request blocked by the safety gate
- save step skipped when no output packet is written

### API And CLI JSON

`generate` summaries and `/api/work-handoff` responses now include `graph_trace`.

This lets future adapters such as Discord, GitHub workflow, or LangGraph-style orchestration reuse the same decision trace without adding autonomous execution.

## What This Is Not

This is not a full LangGraph integration.

It does not:

- call external LLMs
- edit files
- create branches
- create GitHub Issues automatically
- score teammates
- assign work

It is a deterministic local trace layer for visibility.
