# Context Capsule v0.2.8

v0.2.8 is a Guided Result UX release.

It responds to tester feedback that Work Handoff results were technically correct but still too dense for first-time users. The goal is to show what to do first before exposing detailed prompts, graph traces, and raw file candidates.

## Download

```text
context-capsule-v0.2.8.zip
```

## What Changed

- Added `guided_result` to Work Handoff API/CLI summary output.
- Added `추천 첫 행동` at the top of the Work Handoff result.
- Split file candidates into `우선 파일` and `참고 파일`.
- Moved long raw candidate lists behind `전체 후보 자세히 보기`.
- Added portfolio README guidance: root `README.md` is the primary file, nested README files are supporting context.
- Added `--quick` user-speech validation support in v0.2.7 and kept it in release validation flow.

## Why It Matters

The product should not make users read more just to understand what the AI should read.

For example:

```text
요청:
리드미를 포트폴리오용으로 다듬어줘

추천 첫 행동:
root README.md를 기준으로 수정 범위를 잡고,
하위 README는 참고 문서로만 확인하세요.
```

This keeps the result aligned with the product promise:

```text
사람이 판단하기 쉽게 만든 뒤 AI/팀원에게 넘긴다.
```
