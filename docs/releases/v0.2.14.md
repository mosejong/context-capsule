# Context Capsule v0.2.14

v0.2.14 is a Result Trust Flow UI patch.

This release keeps the local rule-based engine intact and focuses on the FastAPI result screen. The goal is to help first-time testers understand what to read first, what evidence was used, and whether conflicting numbers need review before copying a prompt into an AI tool.

## What Changed

- Reordered Work Handoff results into a clearer flow:
  - `요약`
  - `추천 첫 행동`
  - `근거 파일`
  - `충돌/위험`
  - `복붙 프롬프트`
- Added a visible metric-conflict trust card when retrieved documents contain different performance numbers such as `98.6%` and `98.08%`.
- Added a copy button for the AI handoff prompt in the FastAPI UI.
- Updated first-run Korean onboarding copy so testers know exactly what to click and where to look.
- Updated guided-result reading order so API, CLI, and UI speak the same language.

## Why It Matters

Raw agent experiments showed that models can pick stale or conflicting numbers when the repository contains multiple metric values. Context Capsule already detected those conflicts, but the warning was easy to miss inside the risk list.

v0.2.14 turns that into a trust signal:

```text
서로 다른 성능 수치가 발견됐습니다.
검증 문서 기준 수치를 우선 확인하세요.
```

This is not an external LLM feature. It is a local review aid that makes the generated prompt safer to copy.

## What This Is Not

- Not a new retrieval algorithm.
- Not an external Token Analyzer adapter.
- Not a no-terminal desktop installer.
- Not an autonomous code editing agent.

## Validation Target

The expected validation set for this patch is:

```text
pytest
compileall
FastAPI UI static tests
metric conflict API test
```

The key acceptance criterion is that a first-time tester can follow:

```text
요약 -> 추천 첫 행동 -> 근거 파일 -> 충돌/위험 -> 복붙 프롬프트
```
