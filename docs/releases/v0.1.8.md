# Context Capsule v0.1.8

## Release Asset

```text
context-capsule-v0.1.8.zip
```

## Release Type

Dashboard loading and Korean onboarding patch.

v0.1.8 is based on KDT tester feedback after v0.1.7. The core retrieval and security fixes held up, but first-time testers still had two UX problems: they could not immediately find the Korean guide, and after pressing Generate Capsule it was not obvious where to wait for the result.

## What's Improved

- Added `START_HERE_KO.md` at the ZIP root so Korean testers can start without reading the English README first.
- Added a Korean guide link near the top of `README.md`.
- Added Korean dashboard onboarding text that points to `START_HERE_KO.md` and the KDT quickstart.
- Added output-area generation status after `Generate Capsule`:
  - repository scanning
  - request understanding
  - retrieval
  - risk analysis
  - packet generation
- Added result-area error guidance when generation fails.
- Prevented duplicate Generate clicks while a capsule is being generated.
- Cleaned up the Request Understanding expander with Korean labels.
- Kept internal debug details out of Junior/Teammate briefs by using the original task request for user-facing sections.
- Changed Junior/Teammate file lists from raw chunk excerpts to `file path + reason` entries.

## Known Limits

- This is still a local Streamlit dashboard, not a single executable.
- The generation status is UI feedback around the existing synchronous generation service; it is not a background job queue.
- Token numbers remain local estimates (`Estimated only`) until a provider usage or Token Analyzer adapter is connected.
