# Context Capsule v0.1.3

## Release Asset

```text
context-capsule-v0.1.3.zip
```

## Positioning

v0.1.3 is the KDT beta readiness release.

It keeps Context Capsule as a local-first human-in-the-loop handoff tool, then improves the parts first-time testers will actually touch:

- clearer first-run dashboard guidance
- KDT beta quickstart
- expanded real user-speech QA
- retrieval quality metrics in the QA report
- anonymized team-chat style validation cases for v0.2 Scrum/Kickoff direction

## What's New Since v0.1.2

- Added `docs/kdt_beta_quickstart.md` for testers.
- Added Streamlit first-run guide with doctor/index/generate/feedback commands.
- Expanded user-speech QA from 53 to 73 cases.
- Added Discord-style anonymized test phrases:
  - `코랩인데 자꾸 램 터져서 못하겠음`
  - `배경이미지가 검은색으로 나와요`
  - `생성됐다고 나오는데 저장이 안돼요`
  - `timeout 시간 늘려서 다시 테스트하자`
  - `ppt랑 대본이랑 안 맞아요`
  - `화면공유 소리 안 나와요`
- Added QA metrics:
  - hit@1
  - hit@3
  - irrelevant top-path count
  - protected false positives
  - clarification accuracy
- Improved Request Understanding aliases for:
  - runtime/resource issues
  - file save path issues
  - logging issues
  - timeout issues
  - presentation review
  - audio/screen-share issues
  - team coordination/rehearsal phrases
- Added Scrum Notes tests for anonymized runtime and presentation coordination notes.

## Quick Start

Double-click:

```text
run_context_capsule.bat
```

Or use the CLI:

```powershell
.\context_capsule_cli.bat doctor --repo-path .
.\context_capsule_cli.bat index --repo-path . --json
.\context_capsule_cli.bat generate --repo-path . --task "리드미 손보자" --retriever indexed --target all --save --json
.\context_capsule_cli.bat feedback-template --project-name "my-project" --tester-name "nickname" --save --json
```

For KDT testers:

- `docs/kdt_beta_quickstart.md`
- `docs/operations/kdt_beta_test_plan.md`
- `docs/operations/feedback_template.md`

## Safety Notes

- No external AI API is required for the MVP flow.
- GitHub writes stay in dry-run mode unless `--apply` is explicitly used.
- GitHub tokens are read from environment variables only.
- Generated local folders and retrieval indexes are ignored by git.
- Do not paste secrets, `.env`, private API tokens, or proprietary code into feedback files.

## Known Limits

- v0.1.3 is still a Python/Streamlit local app, not a single executable.
- Python 3.13 is required.
- First install needs internet access to install dependencies.
- Retrieval QA is based on local project cases and should keep growing from real tester feedback.
