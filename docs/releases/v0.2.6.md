# Context Capsule v0.2.6

v0.2.6 is a target-positioning and beginner-onboarding release.

It does not add a large new engine feature. It clarifies who Context Capsule is for and how to explain it to people who are not AI-tool specialists.

## Download

```text
context-capsule-v0.2.6.zip
```

## Product Position

Context Capsule is for junior developers who want to use AI coding tools more safely.

It helps answer:

```text
What should the AI look at?
What should the AI not touch?
What does done mean?
What prompt should I copy into Claude, Codex, or ChatGPT?
```

Short explanation:

```text
AI가 아무 파일이나 보고 아무렇게나 고치지 않게,
사람이 먼저 작업 범위를 정리해주는 도구입니다.
```

## Target Users

### Primary User

Junior developers who:

- are unsure which files to show an AI coding tool
- need a safer task brief before asking AI to change code
- want README, issue drafts, meeting notes, and handoff summaries
- want protected areas such as auth, DB, env, deployment, and secrets to stay visible

### Secondary Reader

Interviewers, team leads, and AI beginners who:

- need to understand the product without deep AI vocabulary
- want to see a junior developer's problem definition and verification habits
- care about safe AI use, human approval, and user feedback loops

## What Changed

- README first screen now explains the problem, target user, input/output, and token-reduction idea before technical terms.
- `START_HERE_KO.md` now explains the tool with beginner-friendly Korean and a simple before/after token-flow diagram.
- The FastAPI first screen now states the primary user and secondary reader directly.
- A new [Target Positioning](../target_positioning.md) document records what the product is and what it should not become yet.
- The Korean presentation script now includes a 15-second, 30-second, and interviewer-facing explanation.

## Roadmap Boundary

These are not implemented in v0.2.6:

- image prompt hub
- all-purpose prompt generator
- realtime meeting recording
- realtime STT
- Discord/Zoom automation

They remain future roadmap candidates only. The current focus stays on safe AI coding handoff and text-based meeting/work summaries.

## Tester Focus

Ask testers:

- Can you understand what this tool does within 10 seconds?
- Does the first screen feel aimed at junior developers rather than AI experts?
- Would a team lead or interviewer understand why this project matters?
- Is the token-reduction explanation clear without knowing token internals?
- Is it clear that STT, image prompts, Discord, and Zoom are future ideas rather than current features?
