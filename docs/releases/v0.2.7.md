# Context Capsule v0.2.7

v0.2.7 is a Work Handoff Ownership Check release.

It responds to tester feedback that a generated handoff should help users confirm whether the request belongs to their own part, another teammate's part, or a scope that needs discussion before work begins.

## Download

```text
context-capsule-v0.2.7.zip
```

## What Changed

- Added `내 담당 영역` to Work Handoff in the FastAPI Korean dashboard.
- Added `--my-scope` to CLI `generate`.
- Added `내 파트 확인` to the Work Handoff result tabs.
- Added `ownership_check` to Work Handoff API/CLI/metadata output.
- Added Ownership Check to generated GitHub Issue and Decision Record packets.
- Improved ownership token matching so paths like `README.md` and `app/auth.py` can match user scopes such as `README`, `docs`, `app`, or `auth`.

## Why It Matters

Junior developers often need to ask:

```text
이 작업이 내 파트인가?
다른 사람 파트를 건드릴 가능성이 있나?
회의나 PR에서 먼저 확인해야 하나?
```

Context Capsule now makes that question explicit before a user sends the packet to AI, a teammate, or GitHub Issue.

## Safety Boundary

Ownership Check is not:

- teammate evaluation
- automatic work assignment
- contribution scoring
- automatic branch creation
- automatic code change approval

It is only a confirmation aid. The user still decides whether to proceed.

## Example

```text
내 담당 영역:
README, UI, frontend

요청:
로그인 오류 고쳐줘

결과:
내 파트 확인 -> 확인 필요 또는 다른 파트 가능성
질문 -> 이 작업이 내 담당 범위인지 먼저 확인하세요.
```
