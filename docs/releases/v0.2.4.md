# Context Capsule v0.2.4

v0.2.4 is a tester UX polish release.

v0.2.3 added Workflow Graph Trace. v0.2.4 makes that trace and the surrounding dashboard easier for first testers to understand.

## Download

```text
context-capsule-v0.2.4.zip
```

Download the latest ZIP from:

```text
https://github.com/mosejong/context-capsule/releases/latest
```

Then extract and double-click:

```text
run_context_capsule.bat
```

## What Changed

### Result Reading Order

The Work Handoff summary now tells testers what to read first:

```text
1. 먼저 볼 파일
2. 위험/승인
3. AI 프롬프트
4. 작업 흐름, only when they want to understand why
```

This keeps the new Workflow Graph Trace useful without forcing every tester to read it first.

### Workflow Trace UX Polish

The dashboard now uses Korean stage labels for the current workflow step:

```text
레포 스캔
요청 해석
관련 파일 찾기
위험 확인
작업 패킷 생성
사람 승인 확인
출력 저장
```

Detailed evidence is still available, but it is tucked under `상세 근거 보기` and formatted with Korean labels such as:

```text
요청 의도
확신도
후보 파일
위험도
차단 이유
자동 시작
```

### Feedback Form Expansion

The dashboard and CLI feedback-save command now capture:

- result reading order feedback
- workflow trace feedback

Feedback Review now reports workflow-trace questions separately and can turn repeated confusion into next patch priorities.

## What This Is Not

This release still does not add autonomous agent execution.

It does not:

- call external LLMs
- edit files
- create branches
- create GitHub Issues automatically
- score teammates
- assign work

The product remains local-first and human-in-the-loop.
