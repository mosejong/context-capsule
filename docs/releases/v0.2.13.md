# Context Capsule v0.2.13

v0.2.13 is a Raw vs Capsule Validation and Metric Retrieval patch.

This release closes the v0.2.12 post-release experiment loop. It turns the latest Haiku/Sonnet/Opus comparison work into committed evidence, fixes scoring and retrieval issues found during that experiment, and keeps local-only operation intact.

## What Changed

- Added Raw vs Context Capsule comparison reports across multiple repositories.
- Added an edge-case validation report for vague Korean bug requests and larger repositories.
- Improved Korean colloquial bug handling for requests such as `결제 실패 고쳐줘`, `결제 이상해`, `터져`, and `죽어`.
- Improved metric/QA retrieval so accuracy and performance questions prefer evidence documents such as `docs/`, `reports/`, `qa`, `numbers`, `defense`, and `evaluation`.
- Added Korean synonym scoring for the comparison script so answers like `정확도`, `수치`, `배포`, and `마이그레이션` are not undercounted.
- Moved the optional Anthropic dependency import behind the comparison script runtime path so CI can test local scoring without installing provider SDKs.
- Added `.claude/` to `.gitignore` so local Claude workspace/config files stay out of the public repo.

## Why It Matters

The v0.2.13 experiment makes the product claim sharper:

```text
Context Capsule does not just shorten prompts.
It selects the evidence files that keep smaller models from using stale or wrong context.
```

The strongest example is the Raw vs Capsule comparison:

- Raw mode can still miss or misread the relevant context on larger repositories.
- Context Capsule reduces the candidate context while keeping the key evidence files in scope.
- The comparison script now scores Korean responses more fairly, so the reported quality gap is less distorted by English-only keyword checks.

## What This Is Not

- Not a new external LLM integration.
- Not a Token Analyzer provider adapter.
- Not a no-terminal desktop app.
- Not an automatic code editing agent.

## Upgrade Notes

- Use Python 3.11 or newer.
- The local app and CLI remain external-API-free by default.
- `scripts/run_compare.ps1` remains ignored because it can contain local API keys.
- Raw vs Capsule experiments that call provider APIs should be run intentionally by the user, not as part of normal tests.
