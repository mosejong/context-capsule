# Context Capsule v0.1.5

## Release Asset

```text
context-capsule-v0.1.5.zip
```

## Release Type

Beta hardening release for external repositories and unsafe context handling.

v0.1.5 fixes issues found outside the Context Capsule self-repo demo. The main focus is making rough Korean requests work better against English codebases and preventing untrusted repository text or task text from leaking unsafe instructions and secret-looking values into generated handoff packets.

## What's Fixed

- Added Korean-to-English domain term expansion for common developer requests:
  - `로그인` -> `login`, `auth`, `jwt`, `session`
  - `장바구니` -> `cart`, `basket`, `checkout`
  - `배포` -> `deploy`, `docker`, `compose`, `nginx`
  - plus common terms for mobile, config, env, DB, errors, fixes, tests, and features
- Added external English-codebase retrieval regression tests:
  - Korean login request finds `backend/auth/login.py`
  - Korean cart request finds `frontend/src/cart.js`
  - Korean deploy/config request finds deployment config/docs
- Added context redaction before retrieved file text is used in generated packets:
  - secret-looking values are replaced with `[REDACTED_SECRET]`
  - prompt-injection-like lines are replaced with `[REDACTED_PROMPT_INJECTION_LINE]`
- Added task-request redaction before output folders and saved files are written:
  - secret-looking request text no longer becomes part of the saved output slug
  - generated packet files and `metadata.json` are sanitized before writing
- Added risk gates for redacted unsafe context:
  - redacted secrets become `BLOCKED` findings
  - redacted prompt-injection-like lines become `BLOCKED` findings
  - auto-start is blocked when these findings appear
- Strengthened documentation-only retrieval intent:
  - docs/README requests apply a stronger penalty to code/test files
  - `auth는 건드리지 말고 문서만` stays documentation-oriented while keeping `auth` protected
- Reduced one user-speech QA miss around `issue 생성 명령 에러` by mapping mixed English/Korean GitHub Issue command phrasing to the CLI and GitHub Issue adapter.

## Security Notes

Context Capsule still treats scanned repository text as untrusted input. v0.1.5 adds a redaction layer, but users should still avoid pasting real secrets into task requests, Discord, GitHub Issues, or AI tools.

When `[REDACTED_SECRET]` or `[REDACTED_PROMPT_INJECTION_LINE]` appears in output, treat it as a safety signal. Review the source file locally and do not copy the original unsafe value into external services.

## Quick Start

```powershell
.\context_capsule_cli.bat doctor --repo-path .
.\context_capsule_cli.bat index --repo-path . --json
.\context_capsule_cli.bat generate --repo-path . --task "로그인이 모바일에서만 안돼" --retriever indexed --target all --save --json
```

## Known Limits

- This is still a Python/Streamlit local app, not a single executable.
- First install needs internet access for dependencies.
- Korean-to-English retrieval uses a curated local domain dictionary, not a full multilingual semantic model.
- Redaction is regex-based and intentionally conservative; it can mask token-like text that is not actually a secret.
- Retrieval indexes are local generated state and are not committed.
