# Context Capsule v0.2.11

v0.2.11 is an External Repo Evaluation Harness release.

The main goal is to move external-repo testing from a one-off Claude/manual review into a repeatable script that can fail the build when retrieval or risk behavior regresses.

## What Changed

- Added a fixed external-style FastAPI ecommerce fixture under `tests/fixtures/external_repos/ecommerce`.
- Added 10 realistic task cases in `tests/fixtures/external_repo_eval_cases.json`.
- Added `scripts/evaluate_external_repo.py`.
- Added generated report `docs/reports/external_repo_eval.md`.
- Added regression tests for the evaluation harness.
- Fixed Korean colloquial change intent: `고쳐줘` now helps escalate JWT/auth fix requests to HIGH risk.

## Why It Matters

Context Capsule should not only pass on its own repository.

The new harness checks a separate ecommerce-shaped repo with tasks such as:

- README portfolio polish
- payment fallback work
- JWT expired-token bug fix
- product pagination
- auth service unit test
- DB migration brief
- environment guide
- GitHub Issue draft

The first run caught a real issue: `만료 JWT 토큰 ... 고쳐줘` retrieved the right auth file, but risk stayed MEDIUM. v0.2.11 fixes that so JWT/auth fix intent is treated as HIGH risk.
