# Context Capsule v0.2.5

v0.2.5 is a tester feedback release.

It focuses on two product issues found by real testers:

- explicit file scope must be respected before retrieval ranking
- the default local UI should be understandable to people who are not AI or LLM specialists

## Download

```text
context-capsule-v0.2.5.zip
```

## What Changed

### Explicit File Scope

Context Capsule now detects simple file-scope instructions such as:

```text
전체 폴더의 md파일들을 확인하고 기획서.md를 하나 만들어줘
json은 보지 말고 문서만 정리해줘
```

When a user says `.md files`, first-pass retrieval is constrained to Markdown files. When a user says not to look at JSON, `.json` files are excluded from primary candidates.

This fixes a real tester case where `.json` files appeared under "먼저 볼 파일" even though the user asked to inspect Markdown files.

### Root README Priority

Portfolio README requests now prefer the root `README.md` over nested module README files.

For example:

```text
리드미를 포트폴리오용으로 다듬어줘
```

should start from:

```text
README.md
```

Nested README files can still appear as supporting context, but they should not beat the root README for portfolio-level README work.

### Beginner-Friendly UI Copy

The FastAPI local UI now uses easier labels:

| Previous feel | v0.2.5 label |
| --- | --- |
| LLM | AI |
| keyword | 빠른 검색 |
| hybrid | 균형 검색 |
| indexed | 저장된 검색 |
| packet | 작업 정리본 |
| Task request | 하고 싶은 작업 |
| AI prompt | AI에게 넘길 지시문 |

The search selector also includes a `?` help block explaining when to use each search mode.

### Meeting STT Plan

Scrum Notes Mode remains text-input based in this release.

The docs now record a staged extension plan:

1. paste meeting text
2. upload audio file and convert through a future STT adapter
3. realtime recording/STT after privacy, consent, storage, cost, and performance are designed

No STT API, local STT model, or realtime recording is added in v0.2.5.

## Safety Boundary

v0.2.5 still does not:

- modify files automatically
- create branches automatically
- assign teammates automatically
- score teammate ability
- record meetings
- call external AI services by default

## Tester Focus

Ask testers to check:

- Does `리드미를 포트폴리오용으로 다듬어줘` start from root `README.md`?
- Does `.md 파일만` keep first-pass files to Markdown?
- Does `json은 보지 말고` keep JSON out of primary candidates?
- Is the UI wording easier than earlier versions?
- Is the search mode help text understandable?
