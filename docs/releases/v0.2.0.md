# Context Capsule v0.2.0

## Release Asset

```text
context-capsule-v0.2.0.zip
```

## Release Type

Scrum and Kickoff packet mode release.

v0.2.0 moves Context Capsule beyond single-task handoff. It productizes the text-based collaboration modes so meeting notes and project kickoff ideas can become reviewable execution packets.

## What's Improved

- Scrum Notes Mode now outputs role-discussion questions.
- Project Kickoff Mode now outputs role-discussion questions and safety notes.
- Kickoff out-of-scope always includes:
  - automatic teammate evaluation or scoring
  - automatic assignment without human approval
  - automatic deployment or submission
- GitHub Issue drafts explicitly say not to auto-assign or evaluate teammate skill from meeting text.
- Streamlit dashboard now shows v0.2 safety boundaries in Scrum/Kickoff tabs.
- Streamlit Scrum/Kickoff output tabs include Role Questions.
- Added anonymized Korean meeting fixtures:
  - `tests/fixtures/scrum_runtime_issue_ko.txt`
  - `tests/fixtures/project_kickoff_context_capsule_ko.txt`
- Expanded tests for Scrum/Kickoff fixtures, CLI JSON output, role questions, and safety notes.
- Updated v0.2 docs and demo capture flow.

## Product Boundary

Allowed:

```text
meeting summary
decision extraction
blocker extraction
next-action suggestions
GitHub Issue drafts
role-discussion questions
team-lead review prompts
```

Not allowed:

```text
teammate scoring
automatic role assignment
hidden meeting analysis
automatic branch creation
automatic code execution
automatic deployment or submission
```

Final assignment stays with the human team lead.

## Known Limits

- v0.2.0 uses text input. Discord API integration is not included yet.
- The analyzer is heuristic and local-first; it is not an LLM meeting summarizer.
- Role questions are planning prompts, not people analytics.
