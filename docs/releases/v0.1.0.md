# Context Capsule v0.1.0

First local-app release of Context Capsule.

Context Capsule turns a vague request like "fix this" into a reviewable work packet for AI coding tools, teammates, junior developers, and future-you. It scans a local repository, retrieves task-relevant context, flags risky change areas, estimates token reduction, and can preview a GitHub Issue payload before anything is created.

## Download

```text
context-capsule-v0.1.0.zip
```

Run:

```text
Extract ZIP
Double-click run_context_capsule.bat
Open http://localhost:8501
```

Requirements:

- Windows 10/11
- Python 3.13
- Internet access for first dependency install

## What's Included

- Local Streamlit dashboard
- Windows launcher scripts
- CLI packet generation
- GitHub Issue dry-run/apply adapter
- Saved Markdown/JSON output packets
- AI, teammate, junior, and future-self handoff sections
- Risk analyzer with mention/change risk split
- Local token budget estimate
- Fixed login-error demo scenario
- Performance comparison report and SVG

## Quick CLI Demo

```powershell
.\context_capsule_cli.bat generate --repo-path . --task "Create a login API fix handoff packet" --target all --save --json
.\context_capsule_cli.bat create-issue outputs\YYYYMMDD_HHMMSS_slug --repo mosejong/context-capsule --json
```

`create-issue` is dry-run by default. A real GitHub write requires `--apply`.

## Safety Notes

- MVP features do not require external AI APIs.
- Repository context stays local unless the user explicitly sends it elsewhere.
- GitHub writes require explicit `--apply`.
- GitHub tokens are read from environment variables only.
- `.venv`, `outputs`, `dist`, caches, and credentials are excluded from the release ZIP.

## Known Limits

- v0.1.0 is a Python/Streamlit local app, not a single executable.
- First run installs dependencies into `.venv`.
- Token savings are estimated locally.
- Discord adapter and external Token-analyzer adapter are planned next.
