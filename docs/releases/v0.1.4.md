# Context Capsule v0.1.4

## Release Asset

```text
context-capsule-v0.1.4.zip
```

## Release Type

KDT beta usability hotfix.

This release removes the Python 3.13-only blocker and clarifies first-user behavior around optional indexed retrieval and vague requests.

## What's Fixed

- Relaxed Python requirement from `>=3.13` to `>=3.11`.
- Updated `doctor` so Python 3.11 and 3.12 do not produce a false FAIL.
- Updated Windows installer to search for:
  - `py -3.13`
  - `py -3.12`
  - `py -3.11`
  - `python`
- Clarified that local retrieval indexes are optional:
  - the app still works without an index
  - keyword/path retrieval remains available
  - `index` is recommended when testing `--retriever indexed`
- Improved Streamlit clarification UX:
  - vague requests stop before unrelated retrieval
  - the dashboard shows what detail to add next
  - examples are shown for README, local launcher, and error-log requests

## Quick Start

```powershell
.\context_capsule_cli.bat doctor --repo-path .
.\context_capsule_cli.bat generate --repo-path . --task "리드미 손보자" --target all --save --json
```

Optional indexed retrieval:

```powershell
.\context_capsule_cli.bat index --repo-path . --json
.\context_capsule_cli.bat generate --repo-path . --task "리드미 손보자" --retriever indexed --target all --save --json
```

## Known Limits

- This is still a Python/Streamlit local app, not a single executable.
- First install needs internet access for dependencies.
- Retrieval indexes are local generated state and are not committed.
