# Context Capsule v0.2.1

v0.2.1 is a tester-UX and collaboration-readiness patch.

It moves the default local app from the Streamlit prototype to a Korean-first FastAPI web UI and adds Project Health Check so meeting text can reveal MVP readiness, prototype readiness, missing meeting items, next meeting questions, and ownership confirmation hints.

## Download

```text
context-capsule-v0.2.1.zip
```

Extract the ZIP and double-click:

```text
run_context_capsule.bat
```

Then open:

```text
http://localhost:8501
```

## What's New

- New FastAPI local web UI.
- Korean-first mode cards:
  - 작업 하나 넘기기
  - 회의록 정리
  - 프로젝트 시작 정리
  - 준비도 점검
- Removed the confusing sidebar handoff-target selector from the main tester UI.
- Result area shows a clearer loading state.
- Work Handoff results separate:
  - 요약
  - 관련 파일
  - AI용 프롬프트
  - 팀원용
  - 내일의 나
  - 위험/승인
  - GitHub Issue
- Project Health Check estimates:
  - MVP readiness
  - prototype readiness
  - stability label
  - missing meeting items
  - next meeting questions
  - ownership status
- CLI command added:

```powershell
.\context_capsule_cli.bat health --text "v0.2 UI done. pytest passed." --my-scope "README, UI" --json
```

## Safety Boundaries

- No external LLM is required.
- No teammate scoring.
- No automatic assignment.
- No automatic GitHub Issue creation.
- No automatic branch creation.
- GitHub writes still require explicit `--apply`.

## Why This Patch Exists

Tester feedback showed that the Streamlit sidebar and result tabs were easy to misunderstand:

- `Primary handoff target` looked like it should change the result tabs, but all target tabs were always visible.
- English labels created friction for Korean testers.
- Users wanted to know whether a task belonged to their part or another teammate's part.
- Users needed a clearer first-use path than "pick a sidebar option and inspect several tabs."

v0.2.1 addresses that by making the workflow choice the first screen and keeping the UI text Korean-first.

## Known Limits

- The FastAPI UI is still a local beta UI, not a packaged desktop app.
- Streamlit remains in the repository as a legacy/prototype surface.
- Project Health Check is rule-based and should be read as a planning aid, not an objective score.
- Token Evidence is still `Estimated only`; actual provider usage needs Token Analyzer/provider adapter validation.
