# Context Capsule v0.1.6

## Release Asset

```text
context-capsule-v0.1.6.zip
```

## Release Type

First tester UX patch.

v0.1.6 does not add a new retrieval algorithm. It improves the first 3 minutes for KDT beta testers: where to download, what to click, what to type first, when the terminal is optional, and what feedback to send back.

## What's Improved

- Added a `KDT Testers: Start Here` section near the top of `README.md`.
- Added a direct latest-release link for testers:
  - `https://github.com/mosejong/context-capsule/releases/latest`
- Reframed the first test as dashboard-first:
  - extract ZIP
  - double-click `run_context_capsule.bat`
  - open `http://localhost:8501`
  - type `리드미 손보자`
  - check `Overview`, `AI Handoff Prompt`, and `Risk & Approval`
- Made terminal commands explicitly optional for first-time testers.
- Added a first-run dashboard guide image:
  - `docs/assets/first_run_dashboard_guide.svg`
- Updated Streamlit first-run guidance so the expanded onboarding panel explains the dashboard path before CLI commands.
- Added a Discord-friendly feedback format to `docs/kdt_beta_quickstart.md`.
- Updated release packaging and publish checklist references to v0.1.6.

## Tester Script

Send this to first-time testers:

```text
1. Download the latest ZIP:
   https://github.com/mosejong/context-capsule/releases/latest

2. Extract it.

3. Double-click run_context_capsule.bat.

4. In the dashboard, try:
   리드미 손보자
   로컬 실행 안돼
   auth는 건드리지 말고 문서만 바꾸자

5. Send feedback with:
   request typed / expected files / actual top files / risk result / confusing part
```

## Known Limits

- This is still a Python/Streamlit local app, not a single executable.
- First install needs internet access for dependencies.
- CLI commands remain available, but the recommended first test is dashboard-only.
- The first-run guide image is an onboarding diagram, not a live screenshot of every user's machine.
