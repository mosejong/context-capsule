# Context Capsule v0.1.2

Request Understanding release.

Context Capsule now normalizes real user phrasing before retrieval. It can map rough requests like `리드미 손보자`, `깃헙 이슈 생성 안됨`, `로컬 실행 안돼`, and `토큰 계산 뻥튀기 같은데?` into likely work targets, while keeping protected areas such as `auth` and `db` out of the work scope.

## Download

```text
context-capsule-v0.1.2.zip
```

Run:

```text
Extract ZIP
Double-click run_context_capsule.bat
Open http://localhost:8501
```

Requirements:

- Windows 10/11
- Python 3.13
- Internet access for first dependency install

## What's New Since v0.1.0

- Request Understanding Layer for colloquial Korean task requests
- `indexed` retrieval mode with local persistent JSON index
- visible `retrieval_report` for indexed/fallback behavior
- token baseline changed from whole-repo concat to retrieved candidate file contents
- clarification-only mode for vague requests such as `이거 왜그래?`
- protected-area handling for phrases such as `auth는 건드리지 말고`
- user-speech retrieval QA report
- scanner support for `.bat` and `.ps1` launcher files

## Quick CLI Demo

Build the local retrieval index:

```powershell
.\context_capsule_cli.bat index --repo-path . --json
```

Generate a packet from real user phrasing:

```powershell
.\context_capsule_cli.bat generate --repo-path . --task "리드미 손보자" --retriever indexed --save --json
.\context_capsule_cli.bat generate --repo-path . --task "auth는 건드리지 말고 문서만 바꾸자" --retriever indexed --json
.\context_capsule_cli.bat generate --repo-path . --task "이거 왜그래?" --retriever indexed --json
```

Preview a GitHub Issue payload:

```powershell
.\context_capsule_cli.bat create-issue outputs\YYYYMMDD_HHMMSS_slug --repo mosejong/context-capsule --json
```

`create-issue` is dry-run by default. A real GitHub write requires `--apply`.

## User-Speech Examples

| Input | Expected Behavior |
| --- | --- |
| `리드미 손보자` | Targets `README.md` |
| `심플 리트리버 왜 이럼` | Targets `app/retrievers/simple_retriever.py` |
| `깃헙 이슈 생성 안됨` | Targets `app/cli.py` and `app/adapters/github_issue_adapter.py` |
| `로컬 실행 안돼` | Targets launcher docs/scripts |
| `토큰 계산 뻥튀기 같은데?` | Targets token analyzer and performance report files |
| `auth는 건드리지 말고 문서만 바꾸자` | Targets docs and marks `auth` protected |
| `이거 왜그래?` | Stops with a clarification question |

## Safety Notes

- MVP features do not require external AI APIs.
- Repository context stays local unless the user explicitly sends it elsewhere.
- GitHub writes require explicit `--apply`.
- GitHub tokens are read from environment variables only.
- `.venv`, `outputs`, `dist`, caches, local indexes, and credentials are excluded from the release ZIP.
- If confidence is low, Context Capsule asks one clarification question instead of retrieving repository context.

## Known Limits

- v0.1.2 is a Python/Streamlit local app, not a single executable.
- First run installs dependencies into `.venv`.
- Token savings are estimated locally.
- Request Understanding is rule-based and should keep growing with real user phrasing.
- Discord adapter and external Token-analyzer adapter are planned next.
