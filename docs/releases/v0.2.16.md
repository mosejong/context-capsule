# Context Capsule v0.2.16

v0.2.16 is an explicit folder-scope hotfix based on tester feedback.

The key bug was this case:

```text
my scope: frontend
task: frontend-rn only. Do not look at frontend. Find md files under frontend-rn and polish them for portfolio use.
```

Before this patch, retrieval could still rank unrelated nested README files or let the root README priority override the explicit folder request. It also treated similar prefixes too loosely, so `frontend` could leak into `frontend-rn` work.

## What Changed

- Added explicit include/exclude path hints to request understanding.
- Kept similar path prefixes separate:
  - `frontend-rn` is not the same scope as `frontend`
  - `frontend` can be excluded while `frontend-rn` remains included
- Passed path-scope hints through keyword, hybrid, and indexed retrieval.
- Applied path-scope filtering before ranking candidates.
- Prevented root README portfolio priority from overriding an explicit folder scope.
- Added result guidance for scoped README/portfolio work.
- Added regression tests for the tester case.

## Why It Matters

Context Capsule is supposed to narrow work before AI or teammates act on it.

When a user says:

```text
frontend-rn only
frontend do not read
```

the tool must treat that as a hard work-boundary signal, not as a vague keyword hint.

This patch makes folder scope part of the core request understanding and retrieval contract.

## What This Is Not

- Not a new embedding model.
- Not a Token Analyzer adapter.
- Not a full ownership/permission system.
- Not an external LLM integration.
