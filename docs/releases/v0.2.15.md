# Context Capsule v0.2.15

v0.2.15 is a documentation, screenshot, and release-readiness polish patch.

The core engine stays the same. This release focuses on making the public GitHub page and downloadable ZIP easier to understand for first-time testers, junior developers, and interviewers.

## What Changed

- Added generated demo screenshots to the README:
  - FastAPI local dashboard first screen
  - Work Handoff summary flow
  - Metric conflict/risk card
- Added `scripts/capture_demo_screenshots.py` so screenshots can be regenerated after UI copy/layout changes.
- Reorganized docs into clearer folders:
  - `docs/presentation`
  - `docs/reference`
  - `docs/operations`
  - `docs/archive`
- Added local Markdown link validation so broken internal docs links fail in pytest.
- Updated GitHub Actions runtime versions to avoid the Node.js 20 deprecation warning.
- Updated release packaging/checklist docs for the latest ZIP.

## Why It Matters

Before this patch, the product worked, but the first GitHub impression still relied too much on text. v0.2.15 makes the workflow visible immediately:

```text
first screen
-> result summary
-> evidence files
-> conflict/risk card
```

That matters for portfolio review because an interviewer or tester can understand the product before installing it.

## What This Is Not

- Not a new retrieval algorithm.
- Not a Token Analyzer adapter.
- Not a desktop `.exe` installer.
- Not an external LLM integration.
