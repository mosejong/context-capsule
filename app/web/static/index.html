Context Capsule v0.2.10

AI에게 맡기기 전에 작업 범위를 먼저 정리합니다.

신입 개발자가 “이거 고쳐줘”라고만 던지지 않도록, 먼저 볼 파일과 건드리면 안 되는 범위, 완료 기준과 AI에게 넘길 지시문을 로컬에서 정리합니다. 면접관이나 팀장도 작업 정의와 검증 흐름을 바로 확인할 수 있습니다.

처음이면 이렇게 쓰세요
  1. 대부분은 AI에게 작업 맡기기부터 시작합니다.
  2. 프로젝트 폴더와 평소 AI에게 말하던 요청을 넣습니다.
  3. 추천 첫 행동, 우선 파일, 금지 범위만 먼저 확인합니다.

2차 독자 문제 정의, 위험 통제, 검증 습관을 보고 싶은 면접관/팀장
토큰 절감 원리 레포 전체가 아니라 작업과 관련 있는 파일 후보만 골라 AI 지시문으로 압축합니다.

무엇을 하고 싶나요?

AI에게 코드 작업을 시키기 전

AI에게 작업 맡기기를 선택하세요. 먼저 볼 파일, 내 파트 여부, 위험/승인 항목, AI 지시문을 만듭니다.

회의나 스크럼을 정리할 때

회의록 정리하기를 선택하세요. 결정사항, 막힌 점, 다음 작업, 질문을 뽑습니다.

프로젝트를 처음 시작할 때

프로젝트 시작 준비하기를 선택하세요. MVP 범위, 제외 범위, 제출 체크리스트를 정리합니다.

준비가 충분한지 궁금할 때

준비도 점검을 선택하세요. MVP/프로토타입 준비도와 부족한 회의 항목을 확인합니다.

AI에게 작업 맡기기

“리드미 손보자”, “로그인 안돼”, “로컬 실행 안돼”처럼 입력하세요. 결과는 사람이 다 읽어야 하는 긴 문서가 아니라, AI나 팀원에게 작업 범위를 좁혀주는 정리본입니다.

처음이면 결과에서 요약 → 먼저 볼 파일 → 위험/승인 → AI 지시문 순서만 보면 됩니다.

회의록 정리

스크럼이나 피드백 로그를 붙여넣으면 결정사항, 막힌 점, 다음 작업, 질문을 정리합니다.

프로젝트 시작 정리

아이디어 회의를 MVP 범위, 제외 범위, 작업 흐름, 제출 체크리스트로 정리합니다.

준비도 점검

회의록, 스크럼, 현재 상태를 넣으면 MVP와 프로토타입 준비도를 추정하고, 부족한 회의 항목과 다음 회의 질문을 제안합니다.

피드백 리뷰

저장된 테스터 피드백을 모아서 공통 문제, 다음 패치 우선순위, 회귀 테스트 후보로 정리합니다.
